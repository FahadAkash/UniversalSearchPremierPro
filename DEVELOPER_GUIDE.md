# Developer Guide & API Reference

Universal Search is a highly optimized, real-time CEP extension for Adobe Premiere Pro. It bridges a modern HTML/JS frontend with Premiere's native ExtendScript backend to provide instant, SQL-like querying and batch editing of timeline clips and effects.

## System Architecture

The plugin is strictly divided into two contexts:
1. **Frontend (CEP / Chromium):** Handles the UI, query parsing, filtering, and the 250ms background polling loop.
2. **Backend (ExtendScript):** Interacts directly with Premiere Pro's C++ DOM. Extracts project state into a JSON snapshot and executes timeline mutations (selection, batch edits, playhead movement).

The two communicate via csInterface.evalScript(), which passes serialized JSON back and forth.

## 1. Backend: jsx/hostscript.jsx

This script runs inside Premiere Pro's ExtendScript engine. It is the only file with permission to read or modify the actual Premiere project.

### Core Scanning Methods

* **fs_getActiveState()**
  * **Purpose:** The workhorse of the application. It scans the pp.project.activeSequence, reading every video/audio track, every clip, every effect, and every parameter.
  * **Performance:** Highly optimized. It performs a single-pass loop over clip.components and comp.properties to extract generic properties (Motion, Opacity) and dynamic effect parameters simultaneously.
  * **Output:** Returns a serialized JSON string containing:
    * clips: Array of objects (nodeId, name, type, start/end ticks, effects, effectParams, metadata, offline status).
    * effectParamGroups: A dictionary mapping effect names to their available parameters (used for frontend suggestions).

### Mutation Methods

* **fs_applyBatchEdits(updatesJson)**
  * **Purpose:** Applies bulk modifications to multiple clips instantly.
  * **Supported Edits:** olume (dB), opacity (%), scale (%), otation (degrees), ename (clip name), moveToTrack (track index).
  * **Implementation:** Wraps changes in pp.project.beginUndoGroup() for a single Undo step.

* **fs_selectClips(idsJson)**
  * **Purpose:** Highlights clips in the timeline and moves the playhead.
  * **Note:** Uses a manual clip.setSelected(1, clear) loop instead of sequence.setSelection(), as the latter is buggy and fails to trigger UI highlights in modern Premiere versions.

* **fs_revealProjectItemInTimeline(nodeId)**
  * **Purpose:** Finds every usage of a specific Project Bin item in the active sequence and selects them.

## 2. Frontend: Polling & UI (js/main.js)

This file drives the panel UI and manages the synchronization state.

### State Management
* **pollProject(force)**
  * Runs on a setInterval of **250ms** (POLL_MS).
  * Calls fs_getActiveState(). It hashes the returned JSON string and compares it to the previous hash. If identical, it skips UI rendering to save CPU. If changed, it parses the JSON and updates lastSnapshot.

### Search Execution
* **unSearch(raw, opts)**
  * Triggers whenever the user types in the search bar. Passes the query to query-engine.js to get a filtered subset of lastSnapshot.
  * Calls enderTable(matches) to update the virtual DOM.

### UI Rendering
* **enderCustomSuggestions(query)**
  * Reads the effectParamGroups generated by the backend and populates the auto-complete dropdown with valid parameter names (e.g., ypass, gain) whenever an effect chip is active.

## 3. Frontend: Query Engine (js/query-engine.js)

This module is completely decoupled from the DOM. It takes a raw string and an array of clip objects, returning the matched subset.

### Parsing
* **parseQuery(raw)**
  * Uses regex to extract tokens: key, operator (:, =, <, >, <=, >=), and alue.
  * Evaluates logical NOT (e.g., -effect:blur or !effect:blur).

### Filtering Logic
* Handles strict property filters: mediatype:, duration>, sequence:, 	rack:, status:, label:.
* **Smart Resolver (Dynamic Effect Parameters):**
  * When a user searches for ypass:true, the engine checks the clip's effectParams dictionary.
  * *Disambiguation:* Because multiple effects might have a "Bypass" property, the backend saves them as ypassparametricequalizer. The Smart Resolver looks at the user's active effect:"Parametric Equalizer" chip, prioritizes that context, and intelligently maps the generic ypass search to the specific ypassparametricequalizer property.

## 4. Frontend: Visual Timeline (js/timeline.js)

* **enderTimeline(clips, activeMatches, ...)**
  * Draws the custom interactive timeline at the bottom of the panel using HTML Canvas.
  * Maps the sequence's total duration (end.ticks - zero.ticks) to the pixel width of the window.
  * Overlays purple "match" highlights exactly where the searched clips exist in time.

## Known API Limitations (The "Black Box")

When developing further, be aware of strict limitations in Adobe's ExtendScript API:
1. **Advanced Audio VSTs:** Effects like *Parametric Equalizer*, *Reverb*, and third-party audio plugins expose **zero** properties to the clip.components API. Their parameters (including Bypass) are mathematically impossible to query.
2. **Panner:** The audio Panner is an intrinsic routing feature, not an effect component, and cannot be accessed via script.
3. **Clip Highlighting:** sequence.setSelection([array]) often fails to update the Premiere UI. Always use 	rackItem.setSelected(state, clearSelection) loops instead.
